# US-1.1 — Project Registration

**Slice**: 1 — Technical Foundation  
**Status**: Ready for planning

## User Story

As a developer, I can run `telemetry-client init <server-url>` once per project so that skills and
events can be submitted to the correct server.

## Context

Project registration is the one-time setup step that associates a local project with a server-assigned
`project_id`. The client caches this ID alongside the server URL so subsequent `sync` calls know where
to send data.

The `project_id` is server-assigned — clients do not generate it (ADR-015).  
Developer identity uses `git config user.email` (ADR-002).  
Global hook + per-project opt-in via `init` (ADR-003).

## Scope

**In scope:**
- `telemetry-client init <server-url>` CLI command
- `POST /api/projects` server endpoint
- Local gitignored cache storing `project_id` and `server_url`

**Out of scope:**
- Authentication (MVP has none — ADR-004 / epic-1-scope.md)
- Automated installer for `telemetry-client` (manual install, documented in README)

## API Contract

```
POST /api/projects
Body: { name: string }   ← root directory name of the project
→ 201 { "project_id": string }
```

The `name` field is read from the current working directory name (typically the git repository name).

Server is idempotent: registering the same project name twice creates a new entry each time
(projects are identified by `project_id`, not by name — names are display labels only).

## Client Behavior

1. Reads the project name from the current working directory
2. POSTs `{ name }` to `<server-url>/api/projects`
3. Persists `project_id` and `server_url` to a local gitignored cache
4. If cache already exists: warns user and exits 0 (does not re-register — each `init` call
   creates a distinct `project_id`; re-running would create a new project and orphan historical data)

`telemetry-client sync` exits silently (exit 0) if the local cache is absent — project not initialized.

## Acceptance Criteria

- [ ] `init <server-url>` creates a local cache with `project_id` and `server_url`
- [ ] Project name is derived from the current directory name
- [ ] Running `init` when cache already exists warns and exits 0
- [ ] Server unreachable → stderr error, exit 1
- [ ] `sync` exits 0 silently when cache is absent
