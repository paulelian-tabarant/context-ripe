# US-2.1 — Skill Registration

**Slice**: 3 — Core Pipeline  
**Status**: Ready for planning

## User Story

As the telemetry system, skills present in `.claude/skills/` are registered with the server at
session end, with their assigned IDs cached locally for use in event submission.

## Context

Skill registration is Phase 1 of the `ripe sync` command, which runs via the
`SessionEnd` hook (ADR-001). The server is the single source of truth for skill IDs (ADR-015).
The client maintains a local cache of `(skill_name → skill_id)` mappings. If the cache is lost,
re-registration on the next session recreates it.

Skills are project-scoped: `(project_id, name)` uniquely identifies a skill server-side.

## Scope

**In scope:**
- `POST /api/skills` server endpoint
- Phase 1 of `ripe sync`: read skills, diff against cache, register new ones
- Local cache update with returned `skill_id`s
- Namespaced skill detection and skip

**Out of scope:**
- Skill deletion or archival
- Skills from directories other than `.claude/skills/`

## API Contract

```
POST /api/skills
Body: {
  project_id: string,
  skills: Array<{ name: string }>
}
→ 200 {
  skills: Array<{ name: string, skill_id: string }>
}
```

Server behavior: `ON CONFLICT(project_id, name) DO NOTHING` — same `(project_id, name)` always
returns the same `skill_id`. Only skills absent from the local cache are sent.

## Client Behavior (Phase 1 of `sync`)

1. Reads skill names from `.claude/skills/`
2. Skips skills whose name contains `:` (namespaced / plugin skills) — logs warning to stderr
3. For skills absent from the local cache, POSTs to `POST /api/skills`
4. Writes returned `skill_id`s to the local cache
5. If server is unreachable: logs warning to stderr, exits 0 (non-blocking)

## Acceptance Criteria

- [ ] New skills are registered and their IDs written to the local cache
- [ ] Already-cached skills are not included in the POST payload
- [ ] Same `(project_id, name)` always returns the same `skill_id` (server idempotency)
- [ ] Namespaced skills (containing `:`) are skipped with a stderr warning
- [ ] Server unreachable → stderr warning, continues to exit 0
- [ ] Empty `.claude/skills/` → no POST made
